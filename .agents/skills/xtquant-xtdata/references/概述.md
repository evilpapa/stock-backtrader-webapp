# xtquant.xtdata 概述

来源：[XtQuant.XtData 行情模块](https://dict.thinktrader.net/nativeApi/xtdata.html?id=olYCD5)

## 模块定位

- `xtdata` 是 `xtquant` 中提供行情相关数据的模块。
- 数据来源和权限以 `MiniQmt` 为准；Python 侧通过连接 `MiniQmt` 获取和缓存数据。
- 主要覆盖：
  - 历史和实时行情
  - K 线与分笔
  - 财务数据
  - 合约基础信息
  - 板块和行业分类信息

## 运行逻辑

- 主动获取类接口依赖本地已有数据。
- 若本地数据不足，应先调用 `download_*` 接口补数，再调用 `get_*` 接口读取。
- 订阅类接口通过 `callback` 返回实时数据；同类实时数据通常会进入缓存。
- level2 相关接口依赖终端权限，且通常不保留跨交易日历史。

## 接口分类

- 行情数据：`subscribe_*` / `unsubscribe_*` / `get_*` / `download_*`
- 财务数据：财务表下载与读取
- 基础行情信息：合约详情、板块、指数权重、交易日

## 常用类型说明

### stock_code

- 格式通常为 `code.market`
- 示例：`000001.SZ`、`600000.SH`、`000300.SH`

### period

- level1 常见周期：
  - `tick`
  - `1m`
  - `5m`
  - `15m`
  - `30m`
  - `1h`
  - `1d`
  - `1w`
  - `1mon`
  - `1q`
  - `1hy`
  - `1y`
- 投研版特色数据还支持：
  - `warehousereceipt`
  - `futureholderrank`
  - `interactiveqa`
  - `transactioncount1m`
  - `transactioncount1d`
  - `delistchangebond`
  - `replacechangebond`
  - `specialtreatment`
  - `northfinancechange1m`
  - `northfinancechange1d`
  - `dividendplaninfo`
  - `historycontract`
  - `optionhistorycontract`
  - `historymaincontract`
  - `stoppricedata`
  - `snapshotindex`

### 时间范围

- 请求范围是 `[start_time, end_time]` 闭区间。
- `count > 0` 时表示最多返回最后若干条。
- `count = -1` 通常表示返回全部。
- `start_time=''`, `end_time=''`, `count=-1` 常表示完整范围，但大范围请求会变慢。

### dividend_type

- `none` 不复权
- `front` 前复权
- `back` 后复权
- `front_ratio` 等比前复权
- `back_ratio` 等比后复权

## 请求限制和实践建议

- 单股订阅数量建议不要过多，文档建议尽量不超过 50。
- 标的较多时优先考虑 `subscribe_whole_quote` 全推方案。
- 板块、分类等静态信息更新频率低，适合按日或按周刷新。
- 要解释“为什么没数据”时，优先检查：
  - `MiniQmt` 是否在线
  - 本地是否已下载对应数据
  - 是否具备 level2 权限
  - 时间范围与 `count` 是否冲突
