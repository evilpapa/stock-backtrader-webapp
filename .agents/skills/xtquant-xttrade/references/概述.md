# xtquant.xttrader 概述

来源：[XtQuant.Xttrade 交易模块](https://dict.thinktrader.net/nativeApi/xttrader.html?id=olYCD5)

## 模块定位

- `xttrader` 是 `xtquant` 中负责交易相关能力的模块。
- 它与 `MiniQmt` 交互，负责：
  - 报单
  - 撤单
  - 查询资产、委托、成交、持仓
  - 接收账号、委托、成交、错误等主推回调

## 快速流程

- 创建 `XtQuantTrader(path, session_id)`
- 注册回调 `register_callback(callback)`
- `start()` 启动交易线程
- `connect()` 建立连接
- `subscribe(account)` 订阅账号主推
- 调用下单、撤单、查询接口
- `run_forever()` 阻塞线程接收主推

## 运行逻辑

- `path` 通常指向 `MiniQmt` 安装目录下的 `userdata_mini`。
- `session_id` 是与 `MiniQmt` 通信的会话 ID，不同 Python 策略要避免冲突。
- `connect()` 为一次性连接；断开后不会自动重连。
- 要收到主推，不仅要连接成功，还需要：
  - 注册回调类
  - 订阅对应账号
  - 保持进程持续运行

## 同步与异步

- 同步接口直接返回结果码、订单号或对象。
- 异步接口先返回请求序号 `seq`，后续通过回调推送反馈。
- 在推送回调里调用同步查询时，可能受返回时序影响。

## 宽松时序设置

- `set_relaxed_response_order_enabled(True)` 可以把主动请求返回放到专用线程。
- 优点：避免在推送回调中同步查询时卡死。
- 代价：查询结果和推送消息的时序不再严格串行。
- 文档建议通常优先考虑异步版本查询，而不是依赖宽松时序。

## 常见前置条件

- 股票交易示例通常使用 `StockAccount('账号')`
- 信用业务通常使用 `StockAccount('账号', 'CREDIT')`
- 期货统计等场景可能使用 `StockAccount('账号', 'FUTURE')`
- 市价类型一般只在实盘环境中有效，模拟环境通常不支持市价报单
