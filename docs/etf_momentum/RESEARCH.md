# 全天候动量轮动 ETF 策略设计思路

## 1. 标的资产选择

策略默认使用三类 ETF 构建跨市场、跨资产类别组合：

- `510300`：沪深 300 ETF，代表 A 股核心宽基资产，也是主要基准。
- `513100`：纳指 ETF，代表海外科技成长资产。
- `518880`：黄金 ETF，代表避险资产与通胀对冲工具。

这三类资产分别覆盖国内股票、海外科技和商品/避险资产，相关性相对较低，适合作为动量轮动策略的基础池。

## 2. 动量因子

动量效应的核心假设是“强者恒强”：过去一段时间表现较好的资产，短期内继续表现较好的概率更高。

当前实现使用 20 个交易日窗口：

- 计算过去 20 日日收益率均值作为动量。
- 计算同窗口日收益率标准差作为波动率。
- 用 `动量 / 波动率` 得到风险调整动量。

20 日窗口约等于一个交易月，兼顾趋势捕捉和噪声过滤。

## 3. 风险调整动量

单纯收益动量容易偏向高波动资产。策略引入波动率调整，借鉴短期夏普比率思想，比较单位风险带来的收益。

优势：

1. 不同波动特征的 ETF 可以在同一标准下比较。
2. 自动降低高波动但趋势质量不佳资产的权重。
3. 与正动量筛选共同构成风险控制。

## 4. 权重分配与调仓

- 只选择风险调整动量大于 0 的 ETF。
- 将正风险调整动量归一化为目标权重。
- 若所有 ETF 风险调整动量均不为正，则策略持有现金。
- 当前独立回测脚本默认每日重新计算权重并调仓。

## 5. 数据与回测实现

Python 版本的数据获取已经切换为项目内的 xtdata 客户端：

```python
from utils.xtdata_client import fetch_history_ohlcv, to_title_case_ohlcv

df = to_title_case_ohlcv(fetch_history_ohlcv(ticker, BACKTEST_START, BACKTEST_END))
df = df[["Open", "High", "Low", "Close", "Volume"]].dropna()
```

运行前需要确保本机 miniQMT/xtdata 数据服务可用，并且标的历史行情已准备好。

回测部分使用 Backtrader：

1. 将每只 ETF 的 OHLCV 数据加入 Cerebro。
2. 运行动量策略。
3. 同时运行沪深 300 ETF 买入持有基准和等权组合基准。
4. 输出年化收益率、年化波动率、夏普比率、最大回撤、卡尔马比率、索提诺比率、胜率等指标。
5. 保存每日收益、每日权重和对比图表。

## 6. 风险与局限

- 动量策略在市场快速反转时可能出现回撤。
- 高频调仓会放大交易成本影响。
- 当前回测未单独建模滑点。
- xtdata 数据质量依赖本机 miniQMT 数据下载与服务状态。
- 参数如动量窗口、手续费、标的池需要定期复核。
