# QMT（迅投极速策略交易系统）

QMT（迅投极速策略交易系统）在实际量化应用中，主要分为“内置极速策略模式”和“外部 XtQuant（MiniQMT）模式”。

## 内置极速策略模式（嵌入式运行）

- 运行原理：你的 Python 策略代码直接写在 QMT 客户端内部的编辑器里。当你点击“运行”时，是 QMT 客户端（C++ 主进程）拉起并掌控其内部集成的 Python 解释器来执行你的代码。
- 数据交互：采用进程内（Intra-process）内存直连。底层 C++ 收到行情后，直接通过内部指针传递给 Python 层的 `handle_bar` 回调函数。
- 核心优势：
  - 延迟极低：由于行情数据和交易指令不需要跨进程传输，它拥有原生最低的执行延迟。
  - 开箱即用：界面自带完整的图形化回测报告、资金看板、持仓管理和日志输出。
- 核心劣势：
  - 环境严重受限：Python 版本被券商版 QMT 死死固定（通常是较旧的 3.6/3.8/3.10），无法自由升级。安装第三方科学计算库（如高版本的 TensorFlow、PyTorch 等）极易发生冲突或编译失败。
  - 稳定性绑定：策略与客户端界面在同一个进程中。如果盘中前端界面因为画图卡死或崩溃，你的策略线程也会一同挂掉。

## 外部 XtQuant 模式（解耦/MiniQMT 模式）

- 运行原理：QMT 客户端此时仅仅作为一个“无情的行情与交易网关”在后台默默运行（通常开启极速系统的 Mini 运行模式）。而你的策略代码写在外部独立的 IDE（如 PyCharm、VS Code、Jupyter）里，作为一个完全独立的外部 Python 进程运行。
- 数据交互：采用进程间通信（IPC，底层通常是 Local Socket 或内存映射）。外部的 xtquant 库通过管道向后台的 QMT 终端请求数据或发送订单。
- 核心优势：
  - 环境绝对自由：你可以使用自己的 Anaconda 环境，自由选择 Python 版本（如最新的 Python 3.13），随意安装任何复杂的机器学习或深度学习第三方库，完全不受 QMT 客户端的干扰。
  - 进程安全隔离：即使外部的策略脚本因为代码逻辑 Bug 崩溃了，后台的 QMT 网关依然完好无损，挂单和资金安全可控；反之，界面卡死也不会直接导致策略进程挂掉。
  - 便于架构扩展：非常适合用来写 Adapter（代理），把行情数据转发给自定义的回测框架、Linux 容器或数据库。
- 核心劣势：
  - 延迟略微增加：数据需要跨进程复制传输，虽然在 Windows 本地 IPC 下这个延迟只有微秒到毫秒级，但理论上比内置模式稍慢。
  - 没有原生 UI 界面：失去了 QMT 客户端自带的策略运行图表和标准回测报告，你需要自己打印日志或搭建监控看板（如用 Grafana 或 Streamlit）。
